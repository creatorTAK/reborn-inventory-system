# REBORN 売上分析機能 調査レポート

**作成日**: 2025-12-30
**調査対象**: docs/sales.html（売上管理画面）

## 1. 現状の分析機能一覧

### 1.1 売上分析タブ

売上分析タブでは、選択した期間に対して以下の指標・機能を提供しています。

#### 売上KPIサマリー

| 指標名 | 説明 | 計算方法 |
|--------|------|----------|
| **累計売上高** | 期間内の総売上金額 | `SUM(saleAmount)` |
| **累計利益額** | 期間内の総利益金額 | `SUM(finalProfit)` |
| **利益率** | 売上に対する利益の割合 | `totalProfit / totalRevenue * 100` |
| **累計販売数** | 期間内の販売件数 | `COUNT(販売済商品)` |

#### 在庫KPIサマリー（現在の状態）

| 指標名 | 説明 | 計算方法 |
|--------|------|----------|
| **総出品数** | 現在出品中の商品件数 | `COUNT(status='出品中')` |
| **総在庫数** | 全商品の在庫数量合計 | `SUM(quantity)` |
| **総在庫金額** | 在庫の仕入れ原価合計 | `SUM(quantity * purchasePrice)` |
| **平均単価** | 在庫1点あたりの平均仕入れ価格 | `totalStockValue / totalStock` |

#### グラフ機能

- **売上推移グラフ**: 日別または月別の売上・利益を棒グラフで表示
- **表示切替**: 「日別」「月別」をセレクトボックスで切り替え可能

#### ランキング機能

| ランキング名 | 表示項目 |
|--------------|----------|
| **販売先別売上** | 販売先、販売数、売上、利益 |
| **売上TOP10商品** | 順位、商品名、売上、利益 |

#### 期間フィルタ

- 今月、先月、今年、全期間のプリセット
- 開始日・終了日の自由指定

---

### 1.2 軸分析タブ

軸分析タブでは、多角的な切り口でパフォーマンスを分析できます。

#### 分析軸（8種類）

| 分析軸 | データ元 | 用途 |
|--------|----------|------|
| **ブランド別** | `brand.nameEn` or `brand.nameKana` | どのブランドが稼げるか |
| **大カテゴリ別** | `category.major` | カテゴリ戦略の検討 |
| **中カテゴリ別** | `category.middle` | より詳細なカテゴリ分析 |
| **アイテム名別** | `itemName` | 具体的なアイテム別パフォーマンス |
| **状態別** | `condition.display` | 状態と利益の相関分析 |
| **サイズ別** | `size.display` | サイズ別の需要分析 |
| **販売先別** | `salePlatform` | プラットフォーム戦略 |
| **価格帯別** | 自動分類 | 価格帯ごとの回転・利益 |

#### 分析指標（7種類）

| 指標名 | 説明 | 計算方法 |
|--------|------|----------|
| **仕入数** | 期間内に仕入れた商品数 | `COUNT(purchaseDate in range)` |
| **販売数** | 期間内に販売した商品数 | `COUNT(saleDate in range)` |
| **消化率** | 仕入れに対する販売比率 | `salesCount / purchaseCount * 100` |
| **総売上** | 期間内の総売上金額 | `SUM(saleAmount)` |
| **総利益** | 期間内の総利益金額 | `SUM(finalProfit)` |
| **利益率** | 売上に対する利益の割合 | `finalProfit / salesAmount * 100` |
| **在庫日数** | 出品から販売までの平均日数 | `AVG(saleDate - listingDate)` |

#### ソート機能

- 総利益（高い順/低い順）
- 利益率（高い順/低い順）
- 消化率（高い順/低い順）
- 在庫日数（短い順/長い順）

#### エクスポート機能

- CSVエクスポート対応

## 2. 古着1点もの販売ビジネスに有効な分析指標の提案

業界標準の分析指標とベストプラクティスを調査した結果、以下の指標を追加することで、より精度の高い経営判断が可能になります。

### 2.1 必須で追加すべき指標

#### A. GMROI（Gross Margin Return on Investment）

**定義**: 在庫投資に対する粗利益のリターン

**計算式**: `粗利益 / 平均在庫原価`

**業界ベンチマーク**: 2.5以上が良好

**REBORNでの実装案**:
- 軸分析タブに「GMROI」列を追加
- ブランド別GMROIで「低投資で高リターン」のブランドを特定
- 価格帯別GMROIで最適な仕入れ価格帯を発見

**価値**: 利益率が高くても回転が悪い商品 vs 利益率は普通でも高回転の商品を比較できる


#### B. 滞留日数アラート（Aging Analysis）

**定義**: 出品から一定日数経過した在庫の可視化

**業界ベンチマーク**:
- 2週間経過: 25%値下げ検討
- 4週間経過: 50%値下げ検討
- 6週間経過: 75%クリアランス

**REBORNでの実装案**:
- 在庫一覧に「滞留アラート」バッジ表示
- 売上分析タブに「滞留在庫サマリー」を追加
  - 2週超過: XX件（YY円）
  - 4週超過: XX件（YY円）
  - 6週超過: XX件（YY円）

**価値**: 死蔵在庫を早期発見し、キャッシュフローを改善


#### C. セルスルーレート（Sell-Through Rate）

**定義**: 一定期間内に販売できた商品の割合

**計算式**: `販売数 / (期首在庫 + 仕入数) * 100`

**業界ベンチマーク**: 90日以内に60-80%が健全

**REBORNでの実装案**:
- 売上分析タブに「セルスルーレート」KPIを追加
- 月別セルスルーレートの推移グラフ

**価値**: 仕入れ判断の精度を評価できる


### 2.2 追加推奨指標

#### D. 投資利益率（ROI）

**計算式**: `(売上 - 仕入原価 - 手数料・送料) / 仕入原価 * 100`

**REBORNでの実装案**:
- 軸分析の指標に「投資利益率」を追加
- 利益率との違い: 投資効率を直接測定

**ベンチマーク**: 
- せどり平均: 30-50%
- 古着（状態良好品）: 50-80%


#### E. 季節性分析

**定義**: 季節ごとの売上・利益の変動を分析

**REBORNでの実装案**:
- 売上分析タブに「季節別パフォーマンス」セクション追加
- 過去データから売れやすい季節を商品カテゴリ別に予測

**価値**: 仕入れタイミングの最適化


#### F. プラットフォーム横断分析

**定義**: 同一商品の複数プラットフォームでの価格差・売れやすさ比較

**REBORNでの実装案**:
- 軸分析に「プラットフォーム×カテゴリ」のクロス分析を追加
- 「どのカテゴリをどのプラットフォームで売るべきか」を可視化

**業界データ（参考）**:
- メルカリ平均: ベース価格
- ヤフオク: メルカリ比+14%
- メルカリShops: メルカリ比+21%


### 2.3 指標の優先度と実装推奨順序

| 優先度 | 指標 | 理由 |
|--------|------|------|
| **最高** | GMROI | 在庫投資効率の総合指標 |
| **最高** | 滞留日数アラート | キャッシュフロー改善に直結 |
| **高** | セルスルーレート | 仕入れ精度の検証 |
| **高** | 投資利益率 | 仕入れ判断の基準 |
| **中** | 季節性分析 | 仕入れタイミング最適化 |
| **中** | クロスプラットフォーム分析 | 販売戦略の最適化 |

## 3. オークファン連携との整合性

### 3.1 現在の設計ドキュメント確認

`.serena/memories/AUCFAN_API_INTEGRATION_PLAN.md` を確認しました。

#### 計画されている連携機能

1. **商品登録時の自動価格提案**
   - メルカリ過去取引価格（平均・最高・最低）
   - ヤフオク落札価格
   - 価格推移トレンド
   - 推奨販売価格とレンジ
   - 売れるまでの予測日数

2. **在庫の価値評価・アラート**
   - 在庫価値の可視化
   - 値下げアラート
   - 売り時通知
   - 不良在庫検知

3. **仕入れ判断支援**
   - 市場相場表示
   - 想定利益計算
   - 仕入れ推奨判定

4. **プラットフォーム別価格最適化**
   - 同一商品の価格差分析
   - プラットフォーム別推奨価格

5. **レポート・分析ダッシュボード**
   - 月次収益予測
   - トレンド分析
   - 利益率ランキング
   - 仕入れ推奨レポート


### 3.2 現状のシステムとオークファン連携の整合性

#### 良い点（連携しやすい設計）

| 現在の機能 | オークファン連携との親和性 |
|------------|---------------------------|
| **軸分析（ブランド別）** | ブランド別相場データとの統合が容易 |
| **軸分析（価格帯別）** | 相場データに基づく価格帯分析に拡張可能 |
| **在庫KPI（総在庫金額）** | 市場価値での在庫評価に置き換え可能 |
| **販売先別分析** | プラットフォーム別価格最適化と統合可能 |

#### 改善が必要な点

| 課題 | 対応案 |
|------|--------|
| **相場データ格納場所がない** | 商品データに `marketPrice` フィールドを追加 |
| **価格推移の履歴がない** | `priceHistory` サブコレクションを作成 |
| **滞留在庫の自動検知がない** | オークファンの「値下げアラート」機能と連動 |
| **仕入れ判断のUIがない** | 商品登録画面に「相場チェック」セクションを追加 |

## 4. 結論と推奨アクション

### 4.1 現状の評価

REBORNの売上分析機能は、基本的な売上・利益分析と多軸分析が実装されており、古着せどりビジネスの基盤としては十分機能しています。特に軸分析の8種類の切り口は、ビジネスの多角的な理解に有効です。

### 4.2 短期的な改善推奨（オークファン連携前）

1. **GMROI指標の追加**
   - 軸分析テーブルに列を追加
   - 実装工数: 低

2. **滞留日数アラートの追加**
   - 売上分析タブにサマリーセクション追加
   - 実装工数: 中

3. **セルスルーレートの追加**
   - 売上分析タブのKPIに追加
   - 実装工数: 低

### 4.3 中長期的な改善（オークファン連携後）

1. **市場価値ベースの在庫評価**
2. **値下げ・売り時アラート**
3. **仕入れ推奨レポート**
4. **プラットフォーム別価格最適化**

### 4.4 参考資料

#### 業界ベンチマーク

| 指標 | 古着リセール業界の目安 |
|------|----------------------|
| 在庫回転率 | 年4-6回（2-3ヶ月で完全入れ替え） |
| 利益率 | 20-30%（ブランド品は40%も可） |
| セルスルーレート | 90日以内に60-80% |
| GMROI | 2.5以上 |
| 平均取引額 | $50-75（オンライン） |
